## 0. Purpose

The wheel framework predicts that structured-USP cost (HG-MTH-008.4) and abelian-HSP period-finding cost (HG-MTH-008.5) scale with the multiplicative structure of `n`, especially `omega(n)`, the number of distinct prime factors. HG-MTH-008 also records the arity expression

```text
A_p^2 = 2p/(p-1)^3.
```

This experiment measures both costs across a controlled sweep of squarefree `n` and reports fitted scaling in `omega(n)`, with a separate per-prime arity-feature test. The output is data, not theorem-grade.

---

## 1. Claim discipline

Standard facts used here:

- `omega(n)` is the number of distinct prime factors; for squarefree `n`, this is the number of prime factors.
- For squarefree `n = prod_i p_i`, CRT gives `(Z/nZ)^x ~= prod_i (Z/p_iZ)^x`.
- For `a` coprime to `n`, `ord_n(a) = lcm_i ord_{p_i}(a)`.

Experiment-local placement:

- Operation count is the primary metric. It is deterministic and machine-independent.
- Wall-clock time is secondary and recorded for the measured implementation path.
- Squarefree `n` isolates CRT-factor count from prime-power-depth effects.
- The period-finding operation count is the exact number of modular multiplications a naive order walk would perform, namely `ord_n(a)`. The harness computes this exactly from local orders using the known synthetic factorization so the full sweep is feasible. Optional bounded naive execution can be added later; it is not required for the primary operation-count claim.

Non-claims:

- No complexity-class result.
- No extension to non-squarefree `n`.
- No quantum/QFT measurement claim.
- No P vs NP implication beyond the already fenced factoring corner.

## 2. Experimental design

Default full sweep:

- `omega in {1,2,3,4,5,6}`.
- magnitude bands:
  - `small`: first odd primes beginning with 5;
  - `medium`: random primes in `[10^3, 10^4]`;
  - `large`: random primes in `[10^4, 10^5]`.
- sample size: 30 per `(omega, band)` cell.
- fixed RNG seed.

The full run writes:

```text
data/exp_008_7_results.csv
```

with columns:

```text
seed, omega, magnitude_band, sample_index, n, primes, log2_n, phi_n,
a, ord_a, usp_ops, usp_time_ms, period_ops, period_time_ms,
period_success, period_retries, arity_log_sum
```

## 3. Fitted models

The analyzer fits two ordinary least-squares models for both `usp_ops` and `period_ops` using `log2(cost)` as the response.

Model A:

```text
log2(cost) = c0 + c1*omega + c2*log2(n) + error
```

Model B:

```text
log2(cost) = c0 + c1*omega + c2*log2(n) + c3*sum_i log(2p_i/(p_i-1)^3) + error
```

Report:

- fitted coefficients;
- approximate 95% confidence intervals;
- residual standard error;
- AIC for both models;
- `delta_aic = AIC_B - AIC_A`.

A negative `delta_aic` favors the arity-augmented model. `delta_aic < -10` is treated as meaningful evidence that the arity feature adds explanatory power in this finite sweep.

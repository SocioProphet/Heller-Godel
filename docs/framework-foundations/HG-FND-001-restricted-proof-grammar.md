## 1. Purpose

`HG-FND-001` normalizes the restricted proof-grammar primitive used by the proof-character pipeline.

It supplies the common foundation for:

- the A1 Catalan proof grammar;
- the A2 / P3.a arity-three proof grammar;
- downstream proof-family generating-function work, without taking over `HG-FND-002`;
- grade-chain updates that require the distinction between a typed proof grammar and an untyped tree analogy.

This document discharges the candidate status of the restricted proof-grammar surface. It does not discharge the generating-function construction surface `HG-FND-002` or any Puiseux, phase, or carry-cocycle surface.

## 2. Typed fragment

The framework fragment is a simply typed implicational lambda fragment with constants / constructors.

Types are generated by:

```text
T ::= A | B | ... | (T -> T)
```

Contexts are finite typed environments:

```text
Gamma ::= empty | Gamma, x:T | Gamma, c:T
```

Terms are generated by variables, constants, lambda abstraction, and application:

```text
M ::= x | c | lambda x:T. M | M N
```

The grammar is used only through typed judgments:

```text
Gamma |- M : T.
```

A restricted proof grammar is a typed subgrammar of this fragment obtained by displaying:

1. the target type;
2. the context constants and variables;
3. the allowed production rules;
4. the statistic or statistics used for enumeration.

No untyped constructor, free gauge symbol, analytic branch datum, Puiseux exponent, or external visualization is part of a restricted proof grammar unless it is introduced by an explicit typed production rule.

## 3. Canonical witness convention

Canonical witnesses are alpha-classes of eta-long beta-normal typed lambda terms.

The repository may represent alpha-classes using de Bruijn indices, named variables with alpha-quotienting, or an explicitly equivalent alpha-stable representation. The normative convention is:

```text
witness = alpha-class of eta-long beta-normal inhabitant of the declared target type.
```

Alpha-renaming is quotient data. Beta-reduction and eta-expansion are normalization conventions. They are not additional productions.

A closure document that enumerates witnesses must state whether it enumerates:

1. full eta-long beta-normal lambda terms under a declared statistic; or
2. a constructor-skeleton quotient derived from those normal terms.

The second case is allowed only if the quotient map and statistic relation are explicitly declared.

## 4. Restricted production discipline

A restricted proof grammar is valid only when all witnesses are generated by the displayed productions.

Formally, if a grammar declares a production system `P`, witness family `N_phi`, target type `phi`, and context `Gamma`, then:

```text
s in N_phi  iff  s is generated by P and Gamma |- s : phi,
```

up to the declared alpha / eta-long beta-normal convention.

This is the rule that was previously carried as candidate-`HG-FND-001` in P3.a. It is now normalized here.

## 5. Statistics and declaration discipline

`HG-FND-001` does not require a single universal statistic for every downstream closure. It requires explicit statistic declaration.

A closure document must specify which statistic it uses before assigning a proof-family generating function.

Two statistics are normalized for the A1/A2 grammar surfaces.

### 5.1 Full node-count statistic

The full constructor statistic inherited from the manuscript is:

```text
sigma_C(s)=#lambda-nodes + #application-nodes + #variable-leaves.
```

This statistic is defined on eta-long beta-normal lambda-term representatives.

It is the full node-count statistic. It counts lambda binders, application nodes, and variable leaves in the normal inhabitant.

### 5.2 Constructor-shape statistic

The constructor-shape statistic counts only occurrences of the displayed grammar constructor in the skeleton:

```text
kappa_r(s)=#constructor-nodes in the arity-r skeleton of s.
```

For the A1 binary grammar, `kappa_2` counts occurrences of `b`. For the A2 / P3.a ternary grammar, `kappa_3` counts occurrences of `t`.

Constructor-shape statistics are allowed because P3.b and analogous analytic closures use the constructor-skeleton ordinary generating functions:

```text
C_2(x)=1+x C_2(x)^2
C_3(x)=1+x C_3(x)^3.
```

These are not the same as full node-count generating functions under `sigma_C` unless a change of variables is applied.

### 5.3 Relation between full node count and constructor shape

For the canonical arity-`r` grammar:

```text
C_A^(r) = (A -> ... -> A -> A) -> A -> A
          with r input A slots in the constructor
T ::= x | c(T_1,...,T_r)
where x:A and c:A -> ... -> A -> A.
```

Let `n=kappa_r(T)` be the number of constructor nodes in the skeleton.

The number of variable leaves in the full `r`-ary tree is:

```text
L_r(n)=(r-1)n+1.
```

A constructor occurrence `c(T_1,...,T_r)` is represented in curried STLC syntax by `r` applications:

```text
(((c T_1) T_2) ... T_r).
```

Therefore the number of application nodes contributed by `n` constructor occurrences is:

```text
App_r(n)=r n.
```

For the closed eta-long inhabitant of the target type, the outer binders for `c` and `x` contribute two lambda nodes:

```text
Lam_r(n)=2.
```

Thus, for the canonical A1/A2 family:

```text
sigma_C(T)=Lam_r(n)+App_r(n)+L_r(n)
          = 2 + r n + ((r-1)n+1)
          = 3 + (2r-1)n.
```

For A1 (`r=2`):

```text
sigma_C(T)=3+3n.
```

For A2 / P3.a (`r=3`):

```text
sigma_C(T)=3+5n.
```

Therefore the constructor-shape generating function and full-node generating function are related by a monomial change of variables and an overall shift:

```text
T_r^sigma_C(y)=y^3 C_r(y^(2r-1)).
```

In particular:

```text
T_2^sigma_C(y)=y^3 C_2(y^3)
T_3^sigma_C(y)=y^3 C_3(y^5).
```

This relation preserves the algebraic branch structure under a declared change of variables, while moving the singularity location from `x=rational` in constructor-count coordinates to `y=x^(1/(2r-1))` in full node-count coordinates.

Downstream documents must declare which coordinate they use. P3.b uses constructor-shape coordinate `x`, not full node-count coordinate `y`. P3.a may cite `sigma_C`; P3.b's use of `C_3(x)=1+xC_3(x)^3` is valid because it is explicitly constructor-shape enumeration derived from the same restricted grammar.

## 6. Canonical A1 and A2 grammars

### 6.1 A1 Catalan grammar

Target type:

```text
C_A=(A -> A -> A) -> A -> A.
```

Context:

```text
b:A -> A -> A
x:A
```

Skeleton grammar:

```text
T ::= x | b(T,T).
```

Constructor-shape generating function:

```text
C_2(x)=1+x C_2(x)^2.
```

Full-node generating function under `sigma_C`:

```text
T_2^sigma_C(y)=y^3 C_2(y^3).
```

### 6.2 A2 / P3.a arity-three grammar

Target type:

```text
C_A^(3)=(A -> A -> A -> A) -> A -> A.
```

Context:

```text
t:A -> A -> A -> A
x:A
```

Skeleton grammar:

```text
T ::= x | t(T,T,T).
```

Constructor-shape generating function:

```text
C_3(x)=1+x C_3(x)^3.
```

Full-node generating function under `sigma_C`:

```text
T_3^sigma_C(y)=y^3 C_3(y^5).
```

This normalization explains why the P3.a closure can cite the manuscript `sigma_C` discipline while P3.b can use the constructor-shape equation. They are related statistics on the same normalized restricted grammar, not competing foundations.

## 7. Interface boundary with HG-FND-002

`HG-FND-001` owns:

```text
restricted typed grammar + canonical witness convention + declared statistic discipline.
```

`HG-FND-002` owns:

```text
proof-class / proof-family generating-function construction and analytic extraction discipline.
```

The interface is:

```text
restricted grammar + declared statistic -> countable witness family N_phi -> T_phi^sigma(x)=sum_s x^sigma(s).
```

This document normalizes the left side of the interface. It does not normalize the analytic generating-function construction, coefficient extraction, singularity analysis, or Puiseux data.

## 8. Discharge criterion

A downstream grammar claim may cite normalized `HG-FND-001` when it supplies:

1. target type;
2. typed context;
3. production rules;
4. witness equivalence convention;
5. declared statistic;
6. statement of whether it uses full lambda-term enumeration or constructor-skeleton enumeration;
7. no extra untyped productions.

A grammar that omits these data remains analytic calibration or fixture-grade and may not be cited as theorem-bearing proof-family data.
